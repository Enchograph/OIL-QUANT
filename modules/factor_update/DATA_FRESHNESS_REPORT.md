### 🟡 中等延迟数据源 (3-7天)

#### 7. CFTC (期货持仓) - 延迟: 3-5工作日

| 因子 | 延迟 | 说明 |
|------|------|------|
| WTI_Fut_Vol | 3-5工作日 | 周五收盘，下周二发布 |
| WTI_MM_Net | 3-5工作日 | 同上 |
| WTI_Basis | 可计算 | 需要现货和期货价格 |

**更新规则**:
- CFTC报告每周五收盘后统计
- 报告在下周二发布 (周五 -> 周一 -> 周二)
- 如遇节假日，延迟相应增加

**实际延迟**:
- 周一收到上周五的数据 (延迟3天)
- 周二收到上周五的数据 (延迟4天)
- 周三收到上周五的数据 (延迟5天)

**建议**: CFTC数据适合周度分析，不适合日度高频分析

## 风险提示

### ⚠️ 高延迟风险因子

| 因子 | 延迟 | 风险说明 |
|------|------|----------|
| **US_out** | 1-2月 | 美国原油产量，月频数据 |
| **GPR系列** | 30天 | 地缘政治风险，月内无变化 |
| **TPU系列** | 30天 | 贸易政策不确定性，月内无变化 |
| **Unemployment_Rate** | 30天 | 失业率，月度发布 |
| **M2_Money_Supply** | 30天 | 货币供应量，月度发布 |

**使用建议**:
1. 日度模型中，月频因子使用前向填充 (`ffill()`)
2. 注意区分数据的实际更新频率和填充频率
3. 在模型中考虑延迟因素，避免使用未来信息

### ⚠️ 前视偏差 (Look-ahead Bias) 风险

**危险操作**:
```python
# ❌ 错误：使用未来信息
 today's_prediction = model.predict(today_features)
# 如果 today_features 包含当天收盘后才发布的VIX数据
```

**正确做法**:
```python
# ✅ 正确：使用前一期数据
 today's_prediction = model.predict(yesterday_features)
```

## 结论

### 可实时更新的因子

| 延迟范围 | 数量 | 适用场景 |
|----------|------|----------|
| **T+0 实时** | ~5 | 高频交易 |
| **T+1 日频** | ~48 | 日度预测 |
| **T+3-7 周频** | 8 | 周度分析 |
| **月度** | 122 | 月度策略 |
| **需手动** | 31 | 定期补充 |

**核心结论**:
- ✅ **53个因子**可日度实时更新 (延迟<3天)
- ✅ **61个因子**可周度更新 (延迟<7天)
- ⚠️ **122个因子**是月度数据，月内无变化
- ❌ **31个因子**需手动补充

**建议**:
1. **日度模型**: 使用53个实时因子 + 月频因子(前向填充)
2. **周度模型**: 使用61个周频因子
3. **月度模型**: 使用全部183个可自动化因子
4. **定期手动补充**: OPEC/中国/各国消费产量数据
